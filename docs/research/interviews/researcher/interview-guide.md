# 訪談指引：研究員受訪者

**角色說明**：設計並指派標記任務的受訪者，負責定義標記規則、管理標記員工作進度，並分析標記資料品質。

---

## 訪談前準備

- 確認受訪者有指派過標記任務的經驗
- 準備筆記本或錄音設備（徵得同意後）
- 預計時間：45–60 分鐘

---

## 一、暖身與背景（5–10 分鐘）

| 編號 | 問題 |
|------|------|
| R01 | 請介紹你目前的研究方向，以及標記工作在你的研究中扮演什麼角色？ |
| R02 | 你目前用什麼工具管理標記任務？（Label Studio、自行開發、Excel） |
| R03 | 你大概同時管理幾個工讀生？標記任務的規模大概是多少筆資料？ |
| R04 | 你有使用過 YAML 或設定檔來定義任務嗎？還是偏好視覺化介面？ |

---

## 二、任務設計與建立（15 分鐘）

| 編號 | 問題 |
|------|------|
| R05 | 描述一下你從零設計一個新標記任務的完整流程。 |
| R06 | 你如何將任務分配給工讀生？是你指定，還是讓他們自己選？為什麼？ |
| R07 | 任務的標記規則或說明，你通常怎麼撰寫和傳達給工讀生？ |
| R08 | 在正式啟動標記前，你會先做小規模試標（Pilot）嗎？為什麼？ |
| R09 | 你有試過讓系統自動驗證設定是否正確嗎？還是都靠自己手動確認？ |
| R10 | 任務設定出錯時，通常怎麼發現？會造成什麼損失？ |
| R11 | 如果有一個「試跑模式（Dry Run）」可以讓你在正式啟動前先驗證設定，你覺得有幫助嗎？ |

---

## 三、進度監控與品質管理（15 分鐘）

| 編號 | 問題 |
|------|------|
| R12 | 你目前怎麼追蹤工讀生的標記進度？有沒有視覺化的儀表板？ |
| R13 | 你如何評估工讀生的標記品質？有計算 inter-annotator agreement（IAA）嗎？ |
| R14 | 遇到標記品質不佳時，你怎麼處理？有沒有退件或重標的流程？ |
| R15 | 你最希望在系統上直接看到哪些資料統計或品質指標？ |
| R16 | 資料集品質分析（如類別分布、標記一致性）你目前在哪裡做？（Python 腳本、Excel） |
| R17 | 如果這些分析能直接整合在標記平台裡，會讓你的工作流程改變嗎？如何改變？ |

---

## 四、工讀生管理（10 分鐘）

| 編號 | 問題 |
|------|------|
| R18 | 你如何追蹤每位工讀生的工作量和貢獻？ |
| R19 | 工讀生的薪資計算是你負責嗎？還是有行政人員處理？ |
| R20 | 目前工讀生管理上最花時間的事情是什麼？ |
| R21 | 你希望標記平台能幫你處理哪些工讀生管理的工作？ |

---

## 五、回饋與期望（5–10 分鐘）

| 編號 | 問題 |
|------|------|
| R22 | 你目前用的工具中，最讓你困擾的地方是什麼？ |
| R23 | 如果能重新設計一個符合你需求的標記平台，最重要的三個功能是什麼？ |
| R24 | 你有沒有看過讓你印象深刻的標記或研究管理工具？是什麼讓它做得好？ |
| R25 | 還有什麼事情你想補充，或是你認為我沒問到但很重要的？ |

---

## 訪談後注意事項

- 確認受訪者是否同意錄音及資料使用方式
- 記錄檔命名：`YYYY-MM-DD-researcher-P0X.md`
- 記錄重點：任務設計痛點、對 Dry Run 的態度、品質管理方式、Config YAML 接受度
