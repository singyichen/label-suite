# 訪談指引：Admin（管理者）

**角色說明**：負責帳號管理、工時核對與薪資管理的實驗室行政人員，是連結研究生與工讀生的後台管理角色。

---

## 訪談前準備

- 確認受訪者負責過工讀生管理或薪資相關作業
- 準備筆記本或錄音設備（徵得同意後）
- 預計時間：45–60 分鐘

---

## 一、暖身與背景（5–10 分鐘）

| 編號 | 問題 |
|------|------|
| AD01 | 請介紹你在實驗室中負責的主要職責，工讀生管理佔你工作多大比重？ |
| AD02 | 你目前同時管理幾位工讀生？他們分別在做什麼類型的工作？ |
| AD03 | 你目前用什麼工具管理工讀生帳號、工時和薪資？（Excel、人事系統、自製表單） |
| AD04 | 這些工具是否能滿足你的需求？最讓你困擾的地方是什麼？ |

---

## 二、帳號與權限管理（10 分鐘）

| 編號 | 問題 |
|------|------|
| AD05 | 新工讀生加入時，你的標準作業流程是什麼？帳號如何開設？ |
| AD06 | 目前帳號管理是手動建立還是有系統輔助？遇到什麼問題？ |
| AD07 | 工讀生離職或結束合約時，帳號和資料怎麼處理？ |
| AD08 | 不同角色（工讀生 vs 研究生）的系統權限，你怎麼管理？有沒有遇過權限設定錯誤的問題？ |

---

## 三、工時記錄與核對（15 分鐘）

| 編號 | 問題 |
|------|------|
| AD09 | 你目前用什麼方式取得工讀生的工時資料？（自填表單、系統紀錄、打卡） |
| AD10 | 工時核對的流程是什麼？多久核對一次？大概花多久時間？ |
| AD11 | 有沒有遇過工時資料有誤或有爭議的情況？怎麼解決？ |
| AD12 | 如果系統能自動記錄工讀生在平台上的工作時間，你信任這個數字嗎？還是需要額外核對？ |
| AD13 | 你希望工時報表呈現什麼格式？（每日、每週、每月；個人或彙總） |

---

## 四、薪資計算與發放（15 分鐘）

| 編號 | 問題 |
|------|------|
| AD14 | 薪資計算的基準是什麼？（按時計酬、按件計酬、混合計算） |
| AD15 | 薪資計算目前由誰負責？用什麼工具？手動還是自動化？ |
| AD16 | 薪資計算完成後，工讀生能查看自己的薪資明細嗎？以什麼方式查看？ |
| AD17 | 有沒有遇過薪資計算錯誤的情況？原因是什麼？如何避免？ |
| AD18 | 如果系統能根據工時自動產生薪資計算草稿，你希望有哪些人工確認步驟？ |

---

## 五、任務可見性與資料管控（10 分鐘）

| 編號 | 問題 |
|------|------|
| AD19 | 你需要掌握哪些關於標注任務進度的資訊？（完成率、逾期狀況） |
| AD20 | 不同計畫的資料，工讀生之間有沒有必要相互隔離？有沒有發生過資料混用的情況？ |
| AD21 | 如果工讀生能自己看到所有開放任務並自行選擇，你擔心有什麼問題嗎？ |

---

## 六、回饋與期望（5–10 分鐘）

| 編號 | 問題 |
|------|------|
| AD22 | 你每個月在工讀生管理上花最多時間的事是什麼？最想被系統取代的手動作業是什麼？ |
| AD23 | 如果有一個整合帳號管理、工時記錄、薪資計算的後台，你希望它最重要的功能是什麼？ |
| AD24 | 新帳號採用「邀請制」（管理者發送邀請連結）還是「開放自行註冊」，你偏好哪一種？為什麼？ |
| AD25 | 還有什麼事情你想補充，或是你認為我沒問到但很重要的？ |

---

## 訪談後注意事項

- 確認受訪者是否同意錄音及資料使用方式
- 記錄檔命名：`YYYY-MM-DD-admin-P0X.md`
- 記錄重點：工時記錄信任度、帳號邀請 vs 開放註冊偏好、薪資計算自動化意願
